start Using hexo

Posted on 2018-05-27 Edited on 2019-03-05

今天终于弄一个记录 note 的 blog，选择了github pages 存放静态文件，hexo 作为网站生成器的方案。

选择静态生成而不是有后端的 app 的原因是我这几年习惯用 markdown 记录一些技术的笔记，想着尽可能方便的搬运一些到这个 blog 上。虽然我之前其实给 team 内部搭建一些基于 markdown的文档网站化的时候用的是hugo，这次还是选择了生态稍微好一点的hexo。

站名就叫 思为笔记/Siwei note: note.siwei.info。

我原有的笔记有如下特点:

  • 都是在不同 folder 下边的单个markdown文件加上同级的一个 folder, 内部存放因用到的图片附件。
  • 有一些 LeTaX 的公式
  • 有 [TOC]
  • etc… tbf

我会把迁移的经验更新在这个post 里。
